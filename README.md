# FileSearch - 文件全文搜索索引工具

类似 Archivarius 3000 的桌面文件搜索工具，支持对大容量硬盘进行全文索引和即时搜索。

## 功能特性

- **全文索引**: 索引 doc/docx/xls/xlsx/pdf/ppt/pptx 文件的文本内容
- **即时搜索**: 300ms 防抖，输入即搜索，类似 Everything 的体验
- **离线搜索**: 即使硬盘未连接，仍可通过索引搜索和查看文件内容
- **增量索引**: 只处理新增/修改的文件，基于 mtime + 文件大小判断
- **盘符自适应**: 存储相对路径，移动硬盘更换盘符后仍可正常使用
- **多索引管理**: 同时加载/卸载多个索引文件
- **中文分词**: 使用 jieba 进行中文分词，支持中文关键词搜索
- **关键词高亮**: 搜索结果和预览面板中高亮显示匹配的关键词
- **模糊搜索**: 支持 `*` 通配符模糊匹配
- **系统托盘**: 关闭窗口时最小化到系统托盘
- **自动编码检测**: 自动识别 GBK/UTF-8 等中文编码

## 技术栈

- Python 3.8+
- PyQt5 (GUI)
- SQLite + FTS5 (全文搜索引擎)
- jieba (中文分词)
- chardet (编码检测)

## 安装依赖

```bash
pip install -r requirements.txt
```

## 运行

```bash
python main.py
```

## 使用指南

### 1. 创建索引

1. 点击 **📁 索引管理** 按钮
2. 选择要扫描的目录（如 `E:\`）
3. 选择索引保存位置（默认为程序目录下的 `indexes/` 文件夹）
4. 勾选要索引的文件类型
5. 选择 **增量索引**（推荐）或 **全量重建**
6. 点击 **开始索引**

### 2. 搜索文件

- 在搜索框中输入关键词，自动即时搜索
- 支持 `*` 通配符，如 `报告*` 匹配"报告"开头的文件
- 多个关键词用空格分隔
- 可通过下拉框过滤文件类型和排序方式

### 3. 查看内容

- 点击搜索结果中的文件，右侧预览面板显示文件内容
- 关键词会以黄色高亮显示
- 即使文件所在硬盘未连接，仍可预览索引中保存的内容

### 4. 双击打开文件

- 双击搜索结果可用系统默认程序打开文件
- 如果硬盘未连接，会提示文件不可用，但仍可查看预览

### 5. 系统托盘

- 关闭窗口时程序会最小化到系统托盘
- 双击托盘图标恢复窗口
- 右键托盘菜单可退出程序

## 项目结构

```
FileSearch/
├── main.py                 # 入口文件
├── requirements.txt        # 依赖列表
├── config.ini              # 配置文件（自动生成）
├── indexes/                # 索引文件目录（默认）
├── core/                   # 核心模块
│   ├── database.py         # SQLite 数据库操作
│   ├── indexer.py          # 索引引擎
│   ├── searcher.py         # 搜索引擎
│   ├── text_utils.py       # 文本处理工具
│   └── parsers/            # 文件解析器
│       ├── doc_parser.py   # DOC/DOCX 解析
│       ├── pdf_parser.py   # PDF 解析
│       ├── xls_parser.py   # XLS/XLSX 解析
│       └── ppt_parser.py   # PPT/PPTX 解析
└── ui/                     # 界面模块
    ├── main_window.py      # 主窗口
    ├── index_dialog.py     # 索引管理对话框
    └── preview_panel.py    # 预览面板
```

## 打包为便携版

使用 PyInstaller 打包为单文件可执行程序：

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name FileSearch main.py
```

打包后的可执行文件在 `dist/FileSearch.exe`。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| Ctrl+F | 聚焦搜索框 |
| Escape | 清空搜索 |
| Ctrl+Q | 退出程序 |

## 注意事项

- 首次索引大硬盘（如 1TB）可能需要较长时间，建议使用增量索引
- 索引文件保存在 SQLite 数据库中，可自由复制/移动
- 移动硬盘更换盘符后，程序会自动适配（基于相对路径存储）
- 单个文件最大处理 50MB，PDF 最多处理 200 页
